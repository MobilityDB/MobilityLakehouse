# Examples

## [`mobility-lakehouse-quickstart.ipynb`](https://github.com/MobilityDB/MobilityLakehouse/blob/main/examples/mobility-lakehouse-quickstart.ipynb)

A runnable Jupyter notebook that builds a small mobility lakehouse end to end
with **MobilityDuck** — the engine behind the
[AIS Iceberg Explorer](https://ais-explorer-833836401560.europe-west1.run.app/):
ingest AIS pings → typed trajectories → a **TemporalParquet** shard with covering
columns → a **pruned** query → a lossless round-trip → Iceberg promotion. Every
cell runs against a real MobilityDuck engine; the committed notebook carries its
executed outputs so it renders the full picture on GitHub.

### Run it

1. `pip install jupyter pandas matplotlib`
2. Get a **MobilityDuck-enabled DuckDB** — either `INSTALL mobilityduck FROM
   community; LOAD mobilityduck;` in a DuckDB shell, or build the pinned
   [MobilityDuck](https://github.com/MobilityDB/MobilityDuck) source.
3. Point `MOBILITYDUCK` at that executable and launch Jupyter:

   ```bash
   MOBILITYDUCK=/path/to/duckdb jupyter notebook mobility-lakehouse-quickstart.ipynb
   ```

The notebook is generated by [`_build_notebook.py`](https://github.com/MobilityDB/MobilityLakehouse/blob/main/examples/_build_notebook.py); run
that script to regenerate it.

## [`lake-analytics.ipynb`](https://github.com/MobilityDB/MobilityLakehouse/blob/main/examples/lake-analytics.ipynb)

A second runnable notebook: **temporal analytics over the lake** — what temporal
types give you that plain points cannot. Read straight from a TemporalParquet
shard, it computes per-trajectory length and average speed, *time-travels* to each
vessel's position at a given instant (`valueAtTimestamp`), slices trajectories over
a window (`atTime`), and runs a spatiotemporal query pruned by the covering columns.
Same setup as above; generated by [`_build_analytics_notebook.py`](https://github.com/MobilityDB/MobilityLakehouse/blob/main/examples/_build_analytics_notebook.py).

## [`pruning-at-scale.ipynb`](https://github.com/MobilityDB/MobilityLakehouse/blob/main/examples/pruning-at-scale.ipynb)

A third runnable notebook: **pruning at scale**. It writes a Hive-partitioned
TemporalParquet tree (one shard per day, each with covering columns) and shows,
with `EXPLAIN ANALYZE`, the engine opening only the shards a day-and-region query
needs — partition pruning skips whole files, the covering columns prune row groups
within them, and the Iceberg manifest prunes files at the catalog. Same setup as
above; generated by [`_build_pruning_notebook.py`](https://github.com/MobilityDB/MobilityLakehouse/blob/main/examples/_build_pruning_notebook.py).
